The problem

There are only so many ports that can be opened in Void's firewall. Moreover, specific ports such as 80 and 443 are already in use, and so we would need to server HTTP/S traffic through different ports. This would require you to, for example, access the website using metamechanists.org:8740 (assuming the website was hosted on 8740), which looks terrible, and destroys search engine optimisation, and websites not hosted on port 80/443 are either not indexed or considered unsafe/undesireable.

The solution

This problem is solved with Cloudflare Tunnels. Tunnels connect a network to the rest of the internet without having to open ports on the firewall. This works by nesting a packet inside another packet and one endpoint, and then sending this nested packet to the client through regular methods. The communication between the tunnel and the machine can now be thought of as a browser talking to a web server such as Google, where the machine boths receives and sends data. Once the packet arrives at the destination endpoint, it can be 'un-nested.' On the endpoint that is running on the server machine, the packet is sent to one of several destinations depending on which subdomain the packet came from. For example, a packet from satanicrituals.metamechanists.org may be redirected to port 666, whereas a packet from justinisanoob.metamechanists.org may be redirected to port 999.

Encryption

It should be noted that all traffic between the two endpoints is encrypted. If you need to modify the tunnel, contact Idra as it is entirely done from the Cloudflare web interface.
