Coreprotect

Overview

CoreProtect logs many actions that users do so that we can investigate griefing and other issues. CoreProtect has the capability of rolling back these actions given certain parameters. Crucially, CoreProtect cannot roll back Slimefun items (they will all simply be converted to their vanilla equivalents).


Inspection

/co i

When in inspection mode, you will be shown the logged actions about a block upon clicking it. Left click will show regular actions such as break and destroy, while right click will show transaction history for containers.


Lookup

/co l u:[users] t:[time] r:[radius] a:[action] i:[include] e:[exclude]

Lookup is similar to inspection mode, but far more powerful as it allows you to more directly search CoreProtect's database. Here are some examples.

Find all block break events in the last 5 minutes in a radius of 50 blocks.

/co l r:50 t:5m a:-block

Find all container interaction events in the last 2 weeks in a radius of 5 blocks by the user 'Satan'

/co l r:5 t:2w a:container u:Satan

Find every occurrence of a player killing a bat or creeper in the last 7 minutes and 30 seconds in a radius of 5 blocks

/co l r:5 t:7m,30s a:kill i:creeper,bat

Rollback

/co rollback u:[users] t:[time] r:[radius] a:[action] i:[include] e:[exclude]

Note that the rollback command does not work with Slimefun blocks. These will need to be manually refunded. The format of the command is exactly the same as for lookup. If you do some tomfoolery and unintentionally modify something you didn't mean to, you can use the co restore command.

Roll back all block break events in the last 5 minutes in a radius of 50 blocks.

/co rollback r:50 t:5m a:-block

Roll back all container interaction events in the last 2 weeks in a radius of 5 blocks by the user 'Satan'

/co rollback r:5 t:2w a:container u:Satan

Oops, didn't mean to do that. Undo the rollback that was performed above.

/co restore r:5 t:2w a:container u:Satan
