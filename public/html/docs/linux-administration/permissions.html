Permissions

Linux permissions consist of three main concepts: groups, users, and permission digits. Users are self-explanatory. Groups consist of multiple users and are also fairly self-explanatory. Permission digits are a bit more complicated.

Each file has an owner and a group. Each file also has three permission digits: one for the owner, one for the group, and one for everyone else. Each permission digit shows whether the user can read, write, and execute the file. Permission digits span from 0 to 7, and each type of permission (read, write, and execute) is indicated by 4, 2, and 1 respectively.

For example, a permission digit of 5 is 4+1, and so means 'read and execute.' A permission digit of 3 is 2+1 and so means 'write and execute,' and so on. The permissions of a file can be modified as follows:

chmod [permission-digit] [file] chown [new-owner] [file] chgrp [new-group] [file]

Also, you can list the permissions of all the files in a directory:

ls -l

The setup we use involves an individual user for every service. This makes managing permissions much easier, and every user is by design segregated from all the other users. This in turn means that if one server is compromised, it cannot do any damage to other services without finding a privilege escalation exploit. However, editor accounts do have unrestricted access to all the servers (and several services), and so these accounts must not be compromised.
